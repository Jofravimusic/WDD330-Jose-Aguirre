Learning about data types is very helpul and handy when it comes to talk about data manipulation. Knowing what kind of data we are working, we can have a better code and less errors in the future. At this point, naming variables it is also important, because it will help us to be clear what the variable will do. Declaring a Variable start with the keyword const and it is initialized in that line of the code, because const variables can't be changed in the future. We use let keywork to declare a variable that can be changed while it is manipulated by the code.

When we declare a variable, we need to consider its scope: Global or Local. It is important to be aware of this, because if we declare a local variable to be processed just by a codeblock of a certain function or condicional statement and then we try to use that result outside that scope of the variable, we will have an error in our code. We can store a result from a local variable and pass it to a Global variable. But at this point we need to be carefull and limit us in the use of global scope variables, because we can have problems in the future by having our variable expose to be changed by other function or reassing by another process we can create in our code.

The recommended convention to write names variables is the Camel Case, which starts with a lowercase letter and then each new word is capitalized.

On string manipulation, template literal become so handy when we have to put the name of a variable inside a one line and avoid concatenations of strings.

Learning to manage Arrays, Sets and Maps it is so important, because we can manipulate data and stored it in an advanced way. Using a For Each loop in arrays will help us to access the data stored in it and use it according to the requirements of our program.

As we recognize that arrays are inside the family of Objects, we can rely that objects have some useful methods. Learning to use methods those methods improve how we manage them.

Conditional statements works with boolean results. if - else staments works when a condiction is met and do another if not. When we have just a simple condition, we can use the ternary operator: condition ? (//code to run if condition is true) : (//code to run if condition is false) (pasted from the book)

Functions are the solutions for avoiding code repetition and for this reason, in JavaScript exists different ways to create and evoke it. The most common way to create it is by writting:

function functionName(){ code here}

Other way to declare a function is by assigning it to a variable (Function Expression):

const myFunction = Function(){code here};

A new recent way to declare a function is the arrow function that is written in this way:

const myFunction = () => {code here};

Function Expression and Arrow functions ends with a semicolon, while Functions Literal doesn't need it

When we create a function and we specify data needed to be passed to it, we called it parameters. When we call a function and we pass that data, it is called arguments.
